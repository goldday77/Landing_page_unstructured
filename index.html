⚠ PERHATIAN

Baca dan pahami ketentuan berikut sebelum melanjutkan:

1. Untuk setiap data unstructured yang diterima, misalkan data Informasi Keuangan Daerah berisi 5 file (Excel maupun PDF), maka atas data tersebut dihitung sebanyak 5 dan satuannya ROW.
2. Jika file berbentuk ZIP atau RAR, maka kita extract dulu lalu hitung file yang ada di dalamnya. Ingat, satuannya ROW.

Contoh: 1 RAR berisi 5 PDF → dicatat sebagai 5 rows. Bukan 1 RAR, bukan 1 file, bukan 1 dokumen besar.

Jadi, istilah "row" di sini adalah row administrasi:
 • Bukan row isi data Excel
 • Bukan jumlah baris database
 • Tapi jumlah file/dokumen individual yang diadministrasikan sebagai data unstructured
3. Penamaan: ubah nama folder menjadi nomor tiket, lalu file di dalamnya diubah menjadi nomor tiket + 4 digit nomor urut.
Contoh folder: KM00192972193601263
Nama file: KM001929721936012630001, KM001929721936012630002, dst.
4. Data unstructured wajib disimpan di NAS via Win-SCP. SharePoint hanya boleh jadi media bantu/koordinasi, bukan lokasi final administrasi.
5. Tempat penyimpanan di /E:/2026/UNSTRUCTURED atau /E:/2025/UNSTRUCTURED (sesuaikan tahun terima data, bukan tahun data). Buat folder baru dengan nama "NAMA ILAP_NO NDP".
6. Pengiriman ke PIDE wajib menuliskan direktori tiap jenis data (tidak boleh gelondongan).
Contoh: /E:/2026/UNSTRUCTURED/GOWA ND-651.P_PJ.1031_2026
7. Untuk pengembalian data unstructured, dalam ND pengiriman harus disebutkan: Nomor ND terdahulu · Nama ILAP · Nomor tiket lama · Nomor tiket baru · Jenis data · Jumlah row terdahulu · Jumlah row update · Lokasi direktori.
FAQ
Q: Perlu dihitung ngga rows-nya misalkan file-nya berbentuk Excel?
A: Tidak perlu, langsung dihitung saja sebagai 1 row.
Q: Kalau file-nya ada di beberapa folder bagaimana?
A: Dijadikan satu folder saja (jangan ada subfolder).
Setelah tanggal 1 Juli 2026, tidak ada lagi pengiriman data ke PIDE menggunakan satuan selain ROWS.
Tata Cara Pengelolaan Data Unstructured
Alur Kerja Standar · 9 Langkah
Langkah 1 dari 9
1
Terima Data Unstructured dari ILAP
Pegawai menerima data unstructured dari ILAP (Instansi/Lembaga/Asosiasi dan/atau Pihak Lain). Data dapat berupa file Excel, PDF, Word, ZIP, RAR, atau format lainnya.
📌 Ingat: Catat nama ILAP dan nomor ND (Nota Dinas) pengantar sejak awal. Informasi ini akan dibutuhkan pada langkah-langkah berikutnya.
2
Hitung Jumlah File → Tentukan Jumlah ROW
Teliti data yang diterima dengan menghitung jumlah file satu per satu. Jumlah file tersebut adalah jumlah ROW yang akan direkam.

Jika ada file berbentuk ZIP atau RAR, extract terlebih dahulu, lalu hitung semua file di dalamnya.
# Contoh penghitungan ROW: 1 file Excel → 1 row 1 file PDF → 1 row 1 RAR berisi 5 PDF → 5 rows (bukan 1!) 3 Excel + 2 PDF → 5 rows
⚠ ROW di sini adalah row administrasi = jumlah file/dokumen individual. Bukan jumlah baris dalam Excel, bukan jumlah halaman PDF.
3
Rekam ke Aplikasi DIAMOND
Buka aplikasi DIAMOND dan rekam data unstructured dengan mengisi jumlah ROW sesuai hasil hitungan pada langkah 2.
🔗 Akses DIAMOND:
🖥 http://10.245.4.246:85/diamond/index.php ↗

Isi semua kolom yang tersedia, termasuk nama ILAP, jenis data, dan jumlah ROW.
4
Beri Nama Folder dengan Nomor Tiket DIAMOND
Setelah perekaman berhasil, DIAMOND akan men-generate nomor tiket. Buat folder baru dan beri nama folder tersebut dengan nomor tiket yang dihasilkan.
# Struktur folder: 📁 KM00192972193601263 ← nama = nomor tiket ├── file1.pdf ├── file2.pdf └── file3.xlsx
📌 Penting: Semua file data unstructured dijadikan satu folder. Tidak boleh ada subfolder di dalamnya.
5
Ubah Nama File dengan Nomor Tiket + Nomor Urut
Ubah nama seluruh file di dalam folder menggunakan format: nomor tiket + 4 digit nomor urut mulai dari 0001.
# Format penamaan file: 📁 KM00192972193601263 ├── KM001929721936012630001.pdf ├── KM001929721936012630002.pdf └── KM001929721936012630003.xlsx # Pola: [NomorTiket][0001], [NomorTiket][0002], dst.
💡 Tips: Gunakan aplikasi Rename File yang tersedia di tombol menu untuk mempercepat proses penamaan massal.
6
Buka Win-SCP dan Buat Folder di NAS
Buka aplikasi Win-SCP, navigasi ke direktori NAS sesuai tahun terima data, lalu buat folder baru dengan format NAMA ILAP_Nomor ND Pengantar.
# Direktori tujuan di NAS: /E:/2026/UNSTRUCTURED/ ← jika data diterima tahun 2026 /E:/2025/UNSTRUCTURED/ ← jika data diterima tahun 2025 # Nama folder baru yang dibuat: 📁 GOWA ND-651.P_PJ.1031_2026 ↑ Format: NAMA ILAP_Nomor ND Pengantar
⚠ Gunakan tahun terima data, bukan tahun data itu sendiri.
7
Pindahkan Folder ke NAS via Win-SCP
Upload atau pindahkan folder berisi file data unstructured (yang sudah dinamai dengan nomor tiket) ke dalam folder yang baru dibuat di NAS.
# Struktur akhir di NAS: /E:/2026/UNSTRUCTURED/ └── 📁 GOWA ND-651.P_PJ.1031_2026/ └── 📁 KM00192972193601263/ ├── KM001929721936012630001.pdf ├── KM001929721936012630002.pdf └── KM001929721936012630003.xlsx
📌 Ingat: Data unstructured wajib disimpan di NAS sebagai lokasi final. SharePoint hanya untuk koordinasi/media bantu.
8
Copy Path Directory → Isi Surat Pengantar ke PIDE
Setelah file tersimpan di NAS, copy path directory dari Win-SCP, lalu cantumkan pada keterangan surat pengantar pengiriman data ke Seksi PIDE. Tulis direktori tiap jenis data secara spesifik — tidak boleh gelondongan.
# Contoh penulisan direktori di surat pengantar: Lokasi data: /E:/2026/UNSTRUCTURED/GOWA ND-651.P_PJ.1031_2026
9
Catat di Monitoring Unstructured Data
Setelah data berhasil dikirim ke PIDE, catat seluruh informasi pengiriman pada laman Monitoring Unstructured Data.
🔗 Akses Monitoring:
📊 Buka Monitoring Unstructured Data ↗
✅ RINGKASAN ALUR SELESAI
1. Terima data dari ILAP
2. Hitung ROW (per file)
3. Rekam ke DIAMOND
4. Folder = Nomor Tiket
5. File = Tiket + 0001 dst.
6. Buat folder di NAS
7. Pindahkan ke NAS
8. Copy path → surat PIDE
9. Catat di Monitoring ✅
